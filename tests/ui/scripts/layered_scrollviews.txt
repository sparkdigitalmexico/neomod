# pins wheel SCROLL CHAINING on layered scrollviews (P2.3; was the "UBER hack" pin):
# with the skin dropdown (a UIContextMenu, i.e. a scrollview nested inside
# options_contents) open and hovered, one wheel event produces exactly ONE consumer:
# the dropdown gets first refusal (top-most) but DECLINES because its content fits
# (UIContextMenu disables its own scrolling then), so the wheel chains to the parent
# options_contents -- the page scrolls and the dropdown rides its anchor (the golden
# tail shows the content-movement hover churn). a scrollable dropdown would consume
# instead (pinned by the standalone path, cf. decision log). the trace is identical to
# the pre-P2.3 golden, but by chaining rather than by update-order accident.
# needs the UITestSkin fixture (run.py creates it).
